# Transition Gap — Product Spec v0.1

**작성일**: 2026-05-12
**작성자**: 강훈
**Spec 버전**: v0.1
**개발 단계**: MVP 빌드 중 (5월~6월)

---

## 0. Executive Summary

**Product**: 한국 스타트업 인사제도 진단 → 트레이드오프 시뮬레이션 → 단계적 실행 로드맵 생성 도구

**핵심 통찰**:
> *"AI 시대 조직 재설계의 첫 단계는 인사제도 정합성 회복이다. 누더기 위에 AI를 얹으면 조직 생산성은 오르지 않는다."*

**차별점 4가지**:
1. **"답"이 아니라 "진단과 예측"** — 트레이드오프와 예상 결과만 제시, 결정은 CEO
2. **재무 임팩트 연동** — 한국 스타트업 보상 벤치마크 기반 시뮬레이션
3. **"전환 갭" 프레임 + 가시성 지수** — 시장에 없는 시그니처 IP 2개
4. **단계적 도입 로드맵** — 한 번에 다 바꾸지 않음

**타겟**:
- Primary: 한국 스타트업 30~150명, Series A~B, B2B SaaS / B2C 플랫폼
- Secondary: 한국 중견기업 CHRO

**MVP 완성 목표**: 2026년 5월 31일

## 1. Product 정체성

### 1-1. 두 Layer 분리

| Layer | 정의 | 용도 |
|---|---|---|
| **외부 포지셔닝** | "AI 시대 조직 재설계" | LinkedIn, 콘텐츠 시리즈, 미래 비전 |
| **MVP 실제 범위** | "한국 스타트업 인사제도 정합성 회복" | 7~8월 deploy, case study |

연결 로직: 정합성 회복 = AI 시대 조직 재설계의 첫 단계.

### 1-2. Thesis와의 연결

> *"AI는 인간의 도구가 아니라, 인간이 AI의 도구가 되어간다."*
> *"도메인 시니어가 AI를 무장시키는 것이 차별화 전략."*

Product 자체가 이 thesis의 살아있는 증명. HITL 워크플로우로 도메인 전문가(강훈)가 AI의 출력을 검토/보강하는 구조.

### 1-3. 절대 금지 사항

- 단순 ChatGPT wrapper 금지
- "이 시나리오로 가십시오" 정답 강요 금지
- 한 번에 다 바꾸라는 권고 금지
- 진단 없이 제도부터 추천 금지
- 인건비 임팩트 없는 추천 금지
- 평가/감시 도구 인상 금지
- Workday/Lattice 흉내 금지

## 2. 진단 모듈 (Diagnosis)

### 2-1. 구조

```
Layer 1: 조직 컨텍스트 (5개 변수, 5분)
   ↓
Layer 2: 전환 갭 진단 (21개 변수, 5개 sub-category, 25~35분)
   ↓
가시성 지수 자동 계산
   ↓
진단 리포트 첫 페이지: 가시성 지수 시각화
```

**총 26개 변수, 35~45분 입력**

### 2-2. Layer 1 — 조직 컨텍스트

| ID | 변수 | 선택지 |
|---|---|---|
| L1-1 | 가장 시급한 HR 페인포인트 (최대 2개) | 우수인재 채용 난항 / 핵심인재 이탈 / 인건비 상승 / 평가 공정성 불만 / 사일로 |
| L1-2 | 총 인원수 | 20인 이하 / 20~50 / 50~100 / 100+ |
| L1-3 | 조직 계층 | 3단계 / 4단계 이상 |
| L1-4 | 향후 12개월 채용 기조 | 공격적 확장(30%+) / 결원 보충 / 동결·감축 |
| L1-5 | 산업 도메인 | B2B SaaS / B2C 플랫폼 |

**가중치 규칙**: L1-2 "50인+" + L1-3 "4단계+" → 제도화 시급성 ×1.5

### 2-3. Layer 2 — 전환 갭

#### 2-1. 인력 안정성 갭 (3개 변수)
- 2-1-1 자발적 이직률 / 2-1-2 핵심 인재 이탈 / 2-1-3 조기 퇴사율

**핵심 인재 정의**: *"내일 당장 퇴사 시 진행 중인 핵심 프로젝트가 최소 1개월 이상 지연되는 인력 또는 차세대 리더"*

#### 2-2. 채용 파이프라인 갭 (3개 변수)
- 2-2-1 채용 소요 기간 / 2-2-2 채널 다양성 / 2-2-3 오퍼 거절 빈도

#### 2-3. 총보상 갭 (6개 변수)
- 2-3-1 보상 철학 슬라이더 / 2-3-2 보상 아키타입 (1단계+2단계) / 2-3-3 인건비 비중 / 2-3-4 인건비 증가율 / 2-3-5 시장 대비 위치 / 2-3-6 복리후생 수준

**2-3-2 아키타입 4종**: 현금 안정형 / 단기 성과형 / 장기 비전형 / 혼합형

#### 2-4. 평가 및 수용성 갭 (5개 변수)
- 2-4-1 평가 방식 / 2-4-2 평가-보상 연동 / 2-4-3 CEO vs 직원 공정성 갭 / 2-4-4 비전 공감도 / 2-4-5 평가 운영 데이터 트래킹

**시그니처 wording**: *"블라인드/잡플래닛에서 직원들은 몇 점을 줄 것이라 예상하십니까?"*

#### 2-5. 리더십 및 거버넌스 갭 (6개 변수)
- 2-5-1 리더 부정적 피드백 역량 / 2-5-2 1on1 운영 / 2-5-3 1on1 데이터 트래킹 / 2-5-4 채용 거버넌스 / 2-5-5 배포 거버넌스 / 2-5-6 핵심가치 작동

**시그니처 wording**: *"대표인 내가 직접 나서야 해결됨"* (2-5-1 옵션)

### 2-4. 가시성 지수 (시그니처 IP #1)

**계산 로직**:
- 기본 8개 항목 (항상 분모 포함)
- 조건부 2개 항목 (해당 제도 운영 중일 때만 포함): 2-4-5, 2-5-3
- 분모 = 8 / 9 / 10 동적
- 점수: 측정 중 = 1, 모름 = 0, 보상 구조(2-3-2) 아키타입만 = 0.5

**진단 분기**:
- 60% 미만 → 데이터 가시성 회복 1단계 권고. 정량 시뮬레이션 보류.
- 60% 이상 → 본 진단 진입. 시뮬레이션 신뢰구간 자동 조정.

**시뮬레이션 신뢰구간**:
- 70%+ "신뢰도 높음" / 40~70% "중간" / <40% "정성만"

## 3. 시뮬레이션 모듈 (Simulation)

### 3-1. 3단계 구조

```
Stage 1: 트레이드오프 매트릭스 (As-Is 자동 배치 + To-Be 선택)
   ↓
Stage 2: 시나리오 결과 표시 (도입 제도 + 임팩트 + 재무 + 경고)
   ↓
Stage 3: 절충 인터페이스 (제도 제외 + 도입 시기 조정)
```

### 3-2. Stage 1 — 트레이드오프 매트릭스 (시그니처 IP #2)

**4개 축, 2개 매트릭스로 압축**:

#### 매트릭스 A: Motivation Mix
- X축: 지분/비전 ↔ 현금/인센티브
- Y축: 팀 시너지 ↔ 개인 압도적 성과

| 사분면 | 명칭 |
|---|---|
| Q1 | 단기 성과형 용병조직 |
| Q2 | 장기 비전형 공동체 조직 |
| Q3 | 평균의 함정형 |
| Q4 | 소수정예 중심형 |

#### 매트릭스 B: Operating System
- X축: 자율과 속도 ↔ 통제와 절차
- Y축: 즉시 전력(스킬) ↔ 조직 적합성(컬처핏)

| 사분면 | 명칭 |
|---|---|
| Q1 | 개인플레이어 중심형 |
| Q2 | 가족형 자율 조직 |
| Q3 | 에이전시형 분업/기능 조직 |
| Q4 | 대기업 공채 시스템형 |

#### 풍선 효과 (Balloon Effect) 프레임

> *"두 가치 절충은 가능하지만, 막대한 관리 비용과 고도의 리더십 역량이 요구된다. 한쪽을 누르면 다른 쪽이 튀어나오는 풍선 효과를 인정하고, 조직 체력에 맞는 전략적 편향성을 선택해야 한다."*

#### As-Is 자동 배치 매핑

**매트릭스 A X축**:
- 2-3-2 아키타입 (0.5) / 2-3-2 정량 (0.3, 있을 때만) / 2-3-1 슬라이더 (0.2)

**매트릭스 A Y축**:
- 2-4-2 평가-보상 연동 (0.5) / 2-4-1 평가 방식 (0.2) / L1-1 페인포인트 평균 (0.3)

**페인포인트 절대값**:
- 사일로 0.85 / 평가 공정성 불만 0.65 / 채용 난항 0.5 / 인건비 부담 0.5 / 핵심인재 이탈 0.15

**페인포인트 분산도 보조 지표**: 두 페인포인트 격차 → 매트릭스 시각화 색상에 반영 (0.3+ 시 주황 = 혼돈 신호)

**매트릭스 B X축**:
- 2-5-4 채용 거버넌스 (0.35) / 2-5-5 배포 거버넌스 (0.35) / L1-3 계층 (0.15) / L1-2 인원수 (0.15)

**매트릭스 B Y축**:
- 2-5-6 핵심가치 (0.45) / L1-4 채용 기조 (0.25) / 2-1-3 조기 퇴사율 (0.15) / 2-5-1 리더 피드백 (0.15)

#### 시각화

- As-Is = 흐린 원 (가시성 지수 기반 마커 크기/투명도)
- 가시성 40% 미만 시 경고 메시지
- 페인포인트 분산도 0.3+ 시 주황 톤

### 3-3. Stage 2 — 시나리오 3개

#### 시나리오 1: 성과주의 강화 (Q1 방향)

**도입 제도 패키지**:
- 상대평가 강제분포 (S 10% / A 35% / B 40% / C 15%)
- 차등 인상률 (+20~30% / +5~10% / 0~3% / 동결~-5%)
- PIP 도입
- 상위 인재 RSU/스톡옵션 차등

**예상 임팩트 (12개월)**:
- 핵심 인재 리텐션 ↑ / 사일로 ↑ / 하위 이직 ↑↑ / 평가 갈등 ↑↑

**재무 임팩트**: **인건비 +8~12%**

**필수 경고**:
- 2-2-1 채용 소요 4개월+ → 충원 어려움
- 2-5-1 리더 직접 → PIP 운영 불가
- 2-4-3 공정성 갭 2점+ → 갈등 폭증
- 2-3-3 인건비 50%+ → 성과급 재원 부족
- L1-4 채용 동결 → 시나리오 비권고

**뼈대 제도** (빠지면 시나리오 붕괴): 상대평가, PIP

#### 시나리오 2: 공동체 강화 (Q2 방향)

**도입 제도 패키지**:
- 절대평가 강화 (직급별 역량 기준)
- 인상률 밴드 축소 (0~7%)
- 기본급 비중 상향 (88% → 95%)
- 1on1 의무화
- 스킬 개발 예산 (인당 연 1,000만 원)
- 자율 프로젝트 시간 (주 4시간)

**예상 임팩트**:
- 이직률 ↓ / 응집력 ↑↑ / 단기 생산성 → / 장기 안정성 ↑↑

**재무 임팩트**: **인건비 +7~10% (상향 평준화 + 복리후생 고정비 증가)**

> *"공동체 강화는 인건비 증가율로 보면 성과주의 강화와 큰 차이가 없다. 다만 증가의 성격이 다르다 — 성과주의는 차등 집중, 공동체는 상향 평준화."* (시그니처 인사이트)

**필수 경고**:
- 2-4-2 강한 보상 연동 → 연동 해제 시 리더 저항
- 2-5-4/5 CEO 집중 → 수평성과 거버넌스 불일치
- As-Is가 이미 Q2 근처 → 오버 코스팅

**뼈대 제도**: 절대평가 전환, 인상률 밴드 축소

#### 시나리오 3: Talent Density 집중형 (Q4 방향)

**대상**: Series A 20~50인 구간의 빠른 성장 스타트업

**도입 제도 패키지**:
- 현금: 시장 중위수
- 스톡옵션: 파격적 부여 (시장 상위 25%)
- 마이크로매니징 배제 (완전 자율)
- 컬처핏 + 최상위 역량 채용
- Fast Fire (수습 기간 엄격 검증)

**예상 임팩트**:
- 핵심 인재 리텐션 ↑↑ / 응집력 ↑ / 단기 생산성 ↑

**재무 임팩트**: **인건비 +5~9%** (현금 낮음 + 스톡옵션 분리 회계)

**필수 경고**:
- 2-2-1 채용 소요 4개월+ → 시나리오 불가
- 2-5-6 핵심가치 "문서만" → Fast Fire 법적 리스크
- L1-2 50인+ → 시나리오 적합 범위 초과
- 지분 여유 부족 → 시나리오 무효

**뼈대 제도**: 파격적 스톡옵션, Fast Fire

### 3-4. Q3 탈출 가이드 (As-Is = 평균의 함정형 회사용)

Q3 → Q1/Q2/Q3(Talent Density) 중 선택. 선택 기준:

**Q3 → 시나리오 1 (성과주의)**:
- L1-1에 "사일로" 포함 / 핵심인재 보상 불만 / 채용 파이프라인 견고 / CEO가 "결과 중심" 기질
- 단, 리더십(2-5-1) 준비 먼저

**Q3 → 시나리오 2 (공동체)**:
- L1-1에 "핵심인재 이탈" 포함 / 응집력 높음 / L1-3 "4단계+" / CEO가 "장기 안정성" 기질
- 단, 의사결정 권한 분산 함께

**Q3 → 시나리오 3 (Talent Density)**:
- L1-2 "50인 미만" + 스킬 우선 + 지분 여유

### 3-5. 시나리오 × 매트릭스 B 12개 변형 (적합도 신호등)

| 시나리오 | B-Q1 | B-Q2 | B-Q3 | B-Q4 |
|---|---|---|---|---|
| 1. 성과주의 | 🟢 1-1 실리콘밸리형 | 🟠 1-2 전환 마찰형 (Culture Clash) | 🟢 1-3 KPI 최적화형 | 🟡 1-4 대기업 전환형 |
| 2. 공동체 | 🟠 2-1 느슨한 연합형 (Exodus 경고) | 🟢 2-2 안정 공동체형 | 🟡 2-3 분업 공동체형 | 🟡 2-4 대기업 안정형 |
| 3. Talent Density | 🟢 3-1 정통 Talent Density형 | 🟡 3-2 컬처 중심 정예형 | 🔴 3-3 부적합 → 1번 전환 | 🔴 3-4 부적합 → 2번 전환 |

**신호등**:
- 🟢 자연스러운 결합 — 전환 마찰 최소
- 🟡 주의 요망 — 추가 제도 설계 필수
- 🟠 심각한 마찰 — 기존 조직 해체 각오
- 🔴 도입 불가 — 모순적 결합, 전환 권고

**3-3, 3-4 처리**: 선택은 가능, 빨간 경고 리포트 전체 도배 + 권고 대안 시나리오 안내

### 3-6. Stage 3 — 절충 인터페이스 (MVP 범위 A + B)

**A. 도입 제도 일부 제외**: 체크박스, 뼈대 제도 빼면 시나리오 라벨 변경 ("성과주의 약화형" → "성과주의 시나리오 붕괴")

**B. 도입 시기 조정**: 6 / 12 / 24개월. 6개월 선택 시 빨간 태그 자동 부착 *[초고속 도입: 조직 저항 리스크 높음]*

**C. 재무 상한 설정** → v0.2 보류

**부작용 경고**: Rule-based 트리거 + LLM 텍스트 렌더링 (도메인 환각 방지)

### 5-2. 클라이언트 흐름 vs 강훈 흐름

**클라이언트가 보는 것**:
```
1. 진단 URL 접속 (또는 미팅 동석)
2. 진단 입력 35~45분
3. 안내 문구: "전문가가 N일 내 리포트 전달"
4. → 종료 (결과 웹 조회 불가, 프리미엄 격 유지)
5. PDF 리포트 수령
6. Week 6-7 워크샵 (대면)
7. Week 8 최종 정책 Draft
```

**강훈 백엔드 (1주일)**:
- Streamlit Admin 모드 접속 (.env 비밀번호)
- 세션 진단 결과 확인
- LLM 출력 검토 / 수정 / 회사별 customize
- 재무 임팩트 수동 보강
- PDF 다운로드 / 검토 / 발송

### 5-3. 하이브리드 배포 구조

| 단계 | 위치 | 용도 |
|---|---|---|
| 5월 빌드 | 로컬 (시나리오 A) | 빌더 시그널, MVP 시연 |
